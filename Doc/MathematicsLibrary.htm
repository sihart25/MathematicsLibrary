Object Oriented Software Engineering

 

Maths library assessment

 

9/10/99

Introduction

The aim of this exercise is to create a set of classes that form the basis of a mathematical library.  An object oriented design will be used and the program written in C++.  The library will form a fundamental building block for future assessments, e.g. the Ray Tracer and Ray Caster.

 

The Learning outcomes associated with this project are:

·        Experience in the development of a set of C++ classes

·        Insight into the problems associated with efficient code development in C++

·        Reinforcement of vector and matrix mathematics.

The problem

This exercise is divided into two parts, where the second builds on the first.

 

All your code should be well structured and commented.  Layout is important being both consistent in style and naming conventions.

Part 1 - Vector class

Develop a class Vector that encapsulates the behaviour of a 3D vector i.e. (3, 4, 5).  The vector class should contain at least the following functionality:

Constructors

·        Three doubles

·        Two rays, where the intersection of two rays becomes the new vector (see part 2)

Selectors

·        Individual component selector (e.g. x, y, z)

Streaming out

Streaming in

Mathematics

·        Vector addition

·        Vector subtraction

·        Cross product

·        Dot product

·        Scalar multiplication

·        Distance - Distance between two vectors

·        Magnitude - length of the vector

·        Unit - makes the vector a unit vector

·        Unary operator-() to invert the vector (i.e. v = -1.0 * v )

Part 2 - Ray class

Develop a class Ray that encapsulates the behaviour of an infinite 3D line or ray, i.e. a line without a start and en end.  The class should contain at least the following functionality:

Constructors

·        Two vectors (one representing position, the other direction)

Selectors

·        Component parts

Streaming in

Streaming out

Mathematics

·        Distance of a line to a point

·        Distance of a line to a line

·        Unary operator-() to invert the line (i.e. to invert the direction of the line)

 

Hint:

Consider the ray as being formed from two vectors.

Problem extensions

The following may be added to gain ‘bonus’ marks:

·        Integrate the matrix class, from the exercises, into your vector and ray class library.  Additional functionality will be required to multiply a ray by a matrix and a vector by a matrix.

·        Additional, relevant, functionality in all three classes.

Demonstration

A demonstration of the program may be required.  The demostration should be created in such a way as to test all significant functionality.

Report

An overall project report is required.  This should include:

·        A statement of the problem AND what has been accomplished

·        A brief description of any design decisions

·        Conclusions, problems, etc.

·        Code listing
